# Being VM 运行时选型评估计划

## 背景
同在 (Be With You) · Mirror Mind 需要为 Being 选择一个开源 VM/Agent 框架作为 VM Layer。

## 候选框架
1. OpenClaw
2. AstrBot
3. Claude Code

## 评估矩阵（权重）

| 评估维度 | 权重 |
|----------|------|
| 决策层可替换性 | 30% |
| 持续运行能力 | 20% |
| 执行能力丰富度 | 15% |
| IM 平台接入 | 10% |
| 架构可扩展性 | 10% |
| 社区与维护 | 5% |
| 部署复杂度 | 5% |
| 性能与资源效率 | 5% |

## 执行步骤

### 阶段1：框架研究（各框架并行）
1. **OpenClaw 研究**
   - 架构设计
   - 决策引擎机制
   - 持续运行支持
   - 执行能力（浏览器、文件系统等）
   - IM 接入能力
   - 扩展性机制

2. **AstrBot 研究**
   - 同上维度

3. **Claude Code 研究**
   - 基于当前源码学习项目的理解
   - 补充官方文档信息

### 阶段2：评估打分
- 根据需求规格文档逐项打分
- 计算加权总分

### 阶段3：报告撰写
- 输出选型评估报告
- 给出推荐建议

## 产出
`BEING_VM_EVALUATION_REPORT.md` - 选型评估报告
